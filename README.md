# Blog Platform 🌟

Современная полнофункциональная платформа для ведения блогов, разработанная с использованием актуального стека технологий (MERN). Проект включает в себя полноценную авторизацию, систему комментариев с древовидной структурой ответов, лайки и адаптивный дизайн.

---

## Основные возможности ✨

### 🔐 Авторизация и профиль
- **Регистрация и вход**: Надежная система аутентификации на базе JWT.
- **Приватные маршруты**: Только авторизованные пользователи могут создавать и управлять контентом.
- **Аватары**: Поддержка загрузки и отображения аватаров пользователей.

### 📝 Публикации (CRUD)
- **Создание и редактирование**: Удобный редактор с поддержкой Markdown.
- **Категории**: Классификация постов по различным темам.
- **Удаление**: Возможность безвозвратного удаления своих записей.

### 💬 Социальное взаимодействие
- **Комментарии**: Полноценная система комментирования.
- **Древовидные ответы**: Возможность отвечать на конкретные комментарии с неограниченной вложенностью.
- **Лайки**: Система лайков для постов и комментариев.
- **Счетчик просмотров**: Автоматический учет популярности постов.

### 🎨 Интерфейс и UX
- **Адаптивный дизайн**: Корректное отображение на смартфонах, планшетах и ПК.
- **Микро-анимации**: Плавные переходы и интерактивные элементы.
- **Светлая/Темная тема**: Современная эстетика и удобство чтения.

---

## Технологический стек 🛠️

- **Frontend**: React 18, Redux Toolkit, React Router 6, SCSS, Axios.
- **Backend**: Node.js, Express, Multer (загрузка файлов), JWT.
- **База данных**: MongoDB (Mongoose).
- **Валидация**: express-validator (backend), react-hook-form (frontend).
- **Редактор**: SimpleMDE / EasyMDE.

## Запуск проекта 🚀

### 1. Предварительные требования
- Установленный Node.js (v18+)
- Запущенный экземпляр MongoDB (локально или MongoDB Atlas)

### 2. Настройка Бэкенда
```bash
cd blog_backend
npm install
# Создайте файл .env на основе .env.example (или скопируйте существующий)
cp .env.blog .env 
npm start
```

### 3. Настройка Фронтенда
```bash
cd blog_frontend
npm install
# Настройте прокси или API_URL в константах, если необходимо
npm start
```

Сервер будет доступен по адресу `http://localhost:3000`.
